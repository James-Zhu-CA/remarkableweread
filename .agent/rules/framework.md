# Antigravity Rules Framework (Project Level)

## 1. 语言与沟通 (Language & Communication)
- **文档与建议**: 所有的项目文档 (`.md`)、建议、回复以及研究报告必须使用 **简体中文 (Simplified Chinese)**。
- **代码实现**: 代码中的变量名、函数名、类名以及 **行内注释 (Comments)** 必须使用 **英文 (English)**。
- **思考过程**: Agent 内部推理可以使用英文，但最终展现给用户的输出需遵循上述规则。

## 2. 工作准则 (Principles)
- **简洁性**: 保持回复和日志精炼，直击要点，避免冗余。
- **验证为先**: 完成任务前，务必通过相关工具（如运行脚本、检查文件内容、执行测试）验证结果。
- **Markdown 规范**: 善用 GitHub 风格的 Markdown，使用警告 (Alerts) 突出重要信息。
