# Coding & Naming Standards

## Language
- **Documentation**: Simplified Chinese (简体中文).
- **Code Comments**: English.
- **Commit Messages**: English (recommended) or Chinese.

## Naming Conventions
- **Files**:
    - Use **English** names.
    - Use `snake_case` (e.g., `data_processor.py`) or `kebab-case` (e.g., `deployment-script.sh`).
    - Avoid spaces and special characters.
- **Directories**:
    - `snake_case` or `kebab-case`.

## Formatting
- Keep code clean and well-formatted.
- Use valid Markdown for all documentation.
